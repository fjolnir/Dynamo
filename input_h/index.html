Introduction

A mechanism to define callbacks for keyboard events Does not actually provide a method for listening to the keyboard. Rather you would pipe events from whatever middleware framework or library you are using on your platform

Structs and Unions

_InputObserver

_InputObserver

struct _InputObserver { 
    OBJ_GUTS Input_handler_t handlerCallback; 
    Input_type_t type; 
    unsigned char code; 
    void *metaData;  
    Input_state_t lastKnownState; 
};
Fields
handlerCallback

The function to be called when the event occurs

type

The type of event the observer is interested in

code

The character code the observer is interested in (In case of keyboard events, ignored otherwise)

metaData

An arbitrary pointer for providing context.

Discussion

Observes the event stream for a given event and executes it's handler callback when it encounters it.
