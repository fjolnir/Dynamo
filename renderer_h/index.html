Introduction

Simple 2.5D renderer with a "camera"

The actual drawing of objects is handled by Renderables, whose display callbacks are implemented in their appropriate files.

Discussion

For defining an object you wish to have rendered (You usually wouldn't use this type directly, rather you'd simply add RENDERABLE_GUTS as the first field after the guts of the object you wish to render)

_Renderer

struct _Renderer { 
    OBJ_GUTS GLuint frameBufferId; // The FBO the renderer should draw to 
    vec2_t viewportSize; 
    vec3_t cameraOffset; 
    matrix_stack_t *worldMatrixStack; 
    matrix_stack_t *projectionMatrixStack; 
    LinkedList_t *renderables; // For internal use only 
};
Fields
frameBufferId

The id of the framebuffer the renderer draws to.

viewportSize

The size of the renderer viewport

cameraOffset

The viewing offset of the renderer

worldMatrixStack

The world matrix stack

projectionMatrixStack
